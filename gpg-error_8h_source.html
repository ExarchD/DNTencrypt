49 /* The GnuPG project consists of many components. Error codes are
50  exchanged between all components. The common error codes and their
51  user-presentable descriptions are kept into a shared library to
52  allow adding new error codes and components without recompiling any
53  of the other components. The interface will not change in a
54  backward incompatible way.
55 
56  An error code together with an error source build up an error
57  value. As the error value is been passed from one component to
58  another, it preserver the information about the source and nature
59  of the error.
60 
61  A component of the GnuPG project can define the following macros to
62  tune the behaviour of the library:
63 
64  GPG_ERR_SOURCE_DEFAULT: Define to an error source of type
65  gpg_err_source_t to make that source the default for gpg_error().
66  Otherwise GPG_ERR_SOURCE_UNKNOWN is used as default.
67 
68  GPG_ERR_ENABLE_GETTEXT_MACROS: Define to provide macros to map the
69  internal gettext API to standard names. This has only an effect on
70  Windows platforms.
71 
72  GPGRT_ENABLE_ES_MACROS: Define to provide "es_" macros for the
73  estream functions.
74 
75  In addition to the error codes, Libgpg-error also provides a set of
76  functions used by most GnuPG components. */
